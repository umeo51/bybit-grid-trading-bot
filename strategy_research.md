# 月利5%達成のための取引戦略調査結果

## 目標設定

- **目標月利**: 5%
- **リスク許容度**: 中程度（最大ドローダウン10%以内）
- **取引スタイル**: グリッドトレーディング + 動的リバランス

## 戦略選定：改良型グリッドトレーディング

### なぜグリッドトレーディングか

1. **実績データ**: Bybit公式によると、横ばい相場で週利5%の実績あり
2. **手数料効率**: メイカー注文中心で手数料0.02%
3. **自動化適性**: ロジックが明確で実装しやすい
4. **24時間稼働**: 人間の監視不要

### 月利5%達成の計算

- 週利1.25% × 4週 = 月利5%
- 1日あたり約0.16%の利益が必要
- グリッド1回転あたりの利益: 0.3-0.5%を目標

## 戦略の詳細設計

### 1. 基本パラメータ

| パラメータ | 推奨値 | 理由 |
|:---|:---|:---|
| 取引ペア | BTC/USDT (Perpetual) | 流動性が高く、ボラティリティが適度 |
| 価格範囲 | 現在価格の±3-5% | 1日の変動範囲をカバー |
| グリッド数 | 15-25個 | バランスの良い取引頻度 |
| レバレッジ | 2-3倍 | リスクとリターンのバランス |
| 1グリッドあたりの利益目標 | 0.3-0.5% | 手数料(0.04%)を大きく上回る |

### 2. 動的価格範囲調整

固定レンジではなく、ボラティリティに応じて価格範囲を調整：

```
ATR (Average True Range) を使用
- ATR(14) × 2 を価格範囲の基準とする
- 最小範囲: 現在価格の±2%
- 最大範囲: 現在価格の±8%
```

### 3. リスク管理ルール

1. **ポジションサイズ制限**
   - 総資金の60%まで
   - 1グリッドあたり: 総資金の2-3%

2. **損切りルール**
   - 価格が設定範囲を10%超えて外れた場合、全ポジション決済
   - 日次損失が総資金の5%を超えた場合、取引停止

3. **利益確定ルール**
   - 日次利益が2%に達した場合、新規注文を停止（既存注文は継続）
   - 週次で5%達成した場合、リスクを下げる（グリッド数減少）

### 4. 手数料最適化

- **メイカー注文のみ**: 指値注文で必ずメイカーとなるよう設定
- **注文価格調整**: 現在価格から0.01%以上離れた価格で注文
- **手数料計算式**: 
  ```
  必要利益 = (グリッド幅 × 0.3%) - (往復手数料 0.04%)
  実質利益 = グリッド幅 × 0.26%
  ```

## 実装する機能

### コア機能

1. **グリッド生成エンジン**
   - ATRベースの動的価格範囲計算
   - 等間隔グリッド生成
   - 買い/売り注文の自動配置

2. **注文管理システム**
   - 注文の発注・キャンセル
   - 約定確認と再注文
   - ポジション追跡

3. **リスク管理モジュール**
   - リアルタイム損益計算
   - 自動損切り
   - ポジションサイズ制御

4. **ログ・モニタリング**
   - 取引履歴記録
   - パフォーマンス分析
   - アラート通知

### 補助機能

1. **バックテスト機能**
   - 過去データでの戦略検証
   - パラメータ最適化

2. **ドライラン（紙取引）モード**
   - 実資金を使わずにテスト
