## 現在の設定（300 USDT前提）

### グリッド戦略
- **グリッド数**: 25
- **価格範囲**: ±4%
- **レバレッジ**: 2倍
- **1グリッドあたりの注文サイズ**: 約0.0016 BTC（約150 USDT）

### 計算例（BTC価格 = 96,000 USD、資金 = 300 USDT）

```
総資金: 300 USDT
レバレッジ: 2倍
実質取引可能額: 600 USDT

最大ポジション比率: 50%
最大ポジションサイズ: 300 USDT相当

グリッド数: 25
1グリッドあたり: 300 / 25 = 12 USDT
レバレッジ適用後: 12 * 2 = 24 USDT
BTC換算: 24 / 96000 = 0.00025 BTC... ✗ 最小値未満

実際の計算（最大ポジション比率50%を考慮）:
使用可能資金: 300 * 0.5 = 150 USDT
1グリッドあたり: 150 / 25 = 6 USDT
レバレッジ適用後: 6 * 2 * 2 = 24 USDT
BTC換算: 24 / 96000 = 0.00025 BTC... ✗ 最小値未満

正しい計算:
総資金 * レバレッジ * 最大ポジション比率 / グリッド数
= 300 * 2 * 0.5 / 25
= 12 USDT per grid
= 0.000125 BTC... ✗ 最小値未満
```

**問題**: 上記の計算でも最小注文サイズ（0.001 BTC）に届きません。

### 修正後の設定

グリッド数を**12**に変更します：

```
300 * 2 * 0.5 / 12 = 25 USDT per grid = 0.00026 BTC... まだ不足

300 * 2 * 0.6 / 10 = 36 USDT per grid = 0.000375 BTC... まだ不足

300 * 2 * 0.7 / 8 = 52.5 USDT per grid = 0.000547 BTC... まだ不足

300 * 2 * 0.8 / 6 = 80 USDT per grid = 0.000833 BTC... まだ不足

300 * 2 * 0.9 / 5 = 108 USDT per grid = 0.001125 BTC ✓ クリア！
```

## 最終推奨設定（300 USDT）

```yaml

  count: 5  # グリッド数
  range_percent: 0.03  # 価格範囲 (±3%)

```

この設定で：

- 最小注文サイズをクリア ✓

- 期待月利: 1-2%

## 結論

**Bybitの最小注文サイズ（0.001 BTC = 約96 USDT）が非常に大きいため、少額資金でのグリッド戦略は困難です。**
